Zema plugins are separate packages that build on top of `package:zema`. Each plugin targets one integration layer. The core library has no Flutter or HTTP dependency, plugins add those.

## Available plugins

### zema_forms

Bind schemas to Flutter form widgets. `ZemaFormController` manages per-field validation state and exposes per-field error notifiers so only the field in error rebuilds.

```dart
final ctrl = ZemaFormController(schema: loginSchema);
```

**Package:** `packages/zema_forms`: [reference](./zema_forms)

---

### zema_dio

Validate [Dio](https://pub.dev/packages/dio) HTTP responses against a Zema schema. Two extension methods on `Response`: `parse()` and `safeParse()`.

```dart
final user = response.parse(userSchema);
```

**Package:** `packages/zema_dio`: [reference](./zema_dio)

---

### zema_http

Validate [package:http](https://pub.dev/packages/http) responses against a Zema schema. Decodes JSON and validates in one step. `safeParse()` wraps `FormatException` as a `ZemaFailure` so it never throws.

```dart
final result = response.safeParse(userSchema);
```

**Package:** `packages/zema_http`: [reference](./zema_http)

---

### zema_firestore

Validate Cloud Firestore documents via `withConverter`. Every read is validated, every write converts `DateTime` to `Timestamp`. Uses `withZema()` on `CollectionReference`, `DocumentReference`, and `Query`.

```dart
final usersRef = db.collection('users').withZema(userSchema);
```

**Package:** `packages/zema_firestore`: [reference](./zema_firestore)

---

### zema_hive

Validate Hive documents on read and write. No `TypeAdapter`, no code generation. Uses `withZema()` on `Box`. Supports a `migrate` callback for schema evolution.

```dart
final userBox = box.withZema(userSchema);
```

**Package:** `packages/zema_hive`: [reference](./zema_hive)

---

## Design principles

All plugins follow the same rules:

- The plugin depends on `package:zema`. `package:zema` never depends on the plugin.
- Schemas defined in the core library work unchanged in every plugin.
- Plugins add adapters or widgets. They do not re-implement validation logic.
- Each plugin ships its own `pubspec.yaml` and lives under `packages/`.

## Installing a plugin

Plugins are independent pub packages. Add only what you use:

```yaml
dependencies:
  zema: ^0.4.0
  zema_forms: ^0.1.0       # Flutter form integration
  zema_firestore: ^0.1.0   # Firestore integration
  zema_hive: ^0.1.0        # Hive integration
  zema_dio: ^0.1.0         # Dio HTTP validation
  zema_http: ^0.1.0        # package:http validation
```

Core-only projects (non-Flutter, pure Dart) depend only on `zema`.
