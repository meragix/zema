## Installation

```yaml
dependencies:
  zema: ^0.4.0
  zema_http: ^0.1.0
  http: ^1.2.0
```

```dart
import 'package:http/http.dart' as http;
import 'package:zema/zema.dart';
import 'package:zema_http/zema_http.dart';
```

## Quick start

```dart
final userSchema = z.object({
  'id':    z.string(),
  'name':  z.string().min(1),
  'email': z.string().email(),
});

final response = await http.get(Uri.parse('https://api.example.com/users/1'));

// parse() — returns T or throws ZemaException
final user = response.parse(userSchema);

// safeParse() — returns ZemaResult<T>, never throws
final result = response.safeParse(userSchema);
switch (result) {
  case ZemaSuccess(:final value):
    print(value['name']);
  case ZemaFailure(:final errors):
    print(errors.first.message);
}
```

## How it works

Both methods are extension methods on `http.Response`:

- `parse(schema)`: calls `jsonDecode(body)` then `schema.parse(decoded)`. Throws on invalid JSON or schema mismatch.
- `safeParse(schema)`: calls `jsonDecode(body)` then `schema.safeParse(decoded)`. If `jsonDecode` throws a `FormatException`, it is caught and returned as a `ZemaFailure` with code `invalid_json`. Never throws.

## Handling invalid JSON

`safeParse()` captures `FormatException` from `jsonDecode` and wraps it as a `ZemaFailure`:

```dart
final result = response.safeParse(userSchema);

if (result case ZemaFailure(:final errors)) {
  if (errors.first.code == 'invalid_json') {
    // The response body was not valid JSON
    print('raw body: ${errors.first.receivedValue}');
  }
}
```

## API reference

| Method | Description |
|--------|-------------|
| `response.parse(schema)` | Decode `body` as JSON, validate against `schema`. Returns `T` or throws. |
| `response.safeParse(schema)` | Decode `body` as JSON, validate against `schema`. Returns `ZemaSuccess<T>` or `ZemaFailure`. Never throws. |

## Notes

- Both methods call `jsonDecode(body)` on each invocation. If you need the decoded value separately, decode once and call `schema.parse(decoded)` directly.
- Error handling for HTTP status codes is the caller's responsibility. These methods only validate the body.
- For Dio, use [`zema_dio`](./zema_dio) instead.
