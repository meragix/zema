## Installation

```yaml
dependencies:
  zema: ^0.4.0
  zema_dio: ^0.1.0
  dio: ^5.4.0
```

```dart
import 'package:zema/zema.dart';
import 'package:zema_dio/zema_dio.dart';
```

## Quick start

```dart
final userSchema = z.object({
  'id': z.string(),
  'name': z.string().min(1),
  'email': z.string().email(),
});

final dio = Dio();
final response = await dio.get('/users/1');

// parse(): returns T or throws ZemaException
final user = response.parse(userSchema);

// safeParse(): returns ZemaResult<T>, never throws
final result = response.safeParse(userSchema);
switch (result) {
  case ZemaSuccess(:final value):
    print(value['name']);
  case ZemaFailure(:final errors):
    print(errors.first.message);
}
```

## How it works

Both methods are extension methods on `Response<dynamic>`:

```dart
extension ZemaDioResponseX<E> on Response<E> {
  T parse<T>(ZemaSchema<dynamic, T> schema) => schema.parse(data);
  ZemaResult<T> safeParse<T>(ZemaSchema<dynamic, T> schema) => schema.safeParse(data);
}
```

`data` is the already-decoded response body (Dio decodes JSON automatically). Validation runs directly on the decoded value.

## API reference

| Method | Description |
|--------|-------------|
| `response.parse(schema)` | Validate `response.data` against `schema`. Returns `T` or throws `ZemaException`. |
| `response.safeParse(schema)` | Validate `response.data` against `schema`. Returns `ZemaSuccess<T>` or `ZemaFailure`. Never throws. |

## Notes

- Both methods operate on `response.data`. If Dio is configured to decode JSON, `data` is already a `Map` or `List`. If not, pass a schema that accepts `String` and handles the decoding.
- Error handling for HTTP status codes is Dio's responsibility. These methods only validate the response body once it has been received.
- For `package:http` (not Dio), use [`zema_http`](./zema_http) instead.
